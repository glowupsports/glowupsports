YES — dit screenshot is goud.
Nu weten we exact waarom hij zegt “no internet” terwijl je wél online bent.

Dit is niet Expo, niet OTA, niet branch, niet runtime.

👉 Dit is Android networking + security config.

🔴 WAT ER ECHT MIS GAAT

De keyzin in je screenshot is:

“Download failed (attempt 1/3). Check your network connection.”

Maar je:

✔️ ziet de update UI

✔️ zit online

✔️ preview werkt

❌ download faalt alleen in APK

👉 Dat betekent:

Android blokkeert het download-verzoek zelf

Niet omdat er geen internet is, maar omdat Android het verkeer niet toestaat.

🎯 DE ECHTE OORZAAK (99% zeker)
❌ Android blokkeert de update download omdat:
1️⃣ Cleartext / network security issue

OF

2️⃣ Android mist expliciet netwerk-permissies in de native layer

Expo web/preview → werkt
Android APK → strenger → blokkeert

✅ FIX 1 — ANDROID PERMISSIONS (MOET JE DOEN)

Je hebt dit al deels 👇
Maar we maken het 100% correct.

In app.json → android block

Zorg dat dit erin staat:

"android": {
  "package": "com.glowupsports.app",
  "permissions": [
    "INTERNET",
    "ACCESS_NETWORK_STATE"
  ],
  "usesCleartextTraffic": true
}


👉 usesCleartextTraffic: true is belangrijk, ook al gebruik je HTTPS
(Android is hier heel strict).

✅ FIX 2 — FORCE HTTPS + juiste update URL

Je update-URL is correct, maar we maken ‘m expliciet.

In app.json:

"updates": {
  "enabled": true,
  "checkAutomatically": "ON_LOAD",
  "fallbackToCacheTimeout": 0,
  "url": "https://u.expo.dev/ce3ccb00-0553-4abc-a038-1a93b7483738"
}


👉 NOOIT http
👉 Alleen https://u.expo.dev/...

✅ FIX 3 — REBUILD IS VERPLICHT (dit is native)

⚠️ DIT KAN NIET VIA OTA
Android network settings zitten in de APK.

Dus je MOET doen:
npx eas build --platform android --profile preview


➡️ APK opnieuw installeren
➡️ Daarna werkt OTA-download wél
