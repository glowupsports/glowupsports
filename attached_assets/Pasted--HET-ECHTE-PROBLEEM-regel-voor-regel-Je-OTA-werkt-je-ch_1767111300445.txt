❌ HET ECHTE PROBLEEM (regel voor regel)

Je OTA werkt, je channels kloppen, je builds kloppen
👉 maar je app.json mist 1 cruciale OTA-regel, waardoor updates wel worden gedownload maar NIET worden toegepast.

1️⃣ Je huidige updates block is ONVOLLEDIG

Je hebt nu:

"updates": {
  "url": "https://u.expo.dev/ce3ccb00-0553-4abc-a038-1a93b7483738"
}


❌ Wat hier mist:

enabled

checkAutomatically

fallbackToCacheTimeout

Zonder die defaults kiest Expo vaak de embedded bundle
→ precies wat jij ziet: “downloaded maar geen update zichtbaar”

✅ DE FIX (DIT MOET JE DOEN)
🔧 Vervang je updates block door dit:
"updates": {
  "enabled": true,
  "checkAutomatically": "ON_LOAD",
  "fallbackToCacheTimeout": 0,
  "url": "https://u.expo.dev/ce3ccb00-0553-4abc-a038-1a93b7483738"
}

Wat dit oplost:

✅ App checkt altijd bij openen

✅ OTA krijgt prioriteit boven embedded JS

✅ Geen “oude UI blijft hangen”

2️⃣ Je runtimeVersion is GOED (geen fout hier)

Dit is correct 👇

"runtimeVersion": {
  "policy": "appVersion"
}


➡️ Matches perfect met je builds (1.0.0)
➡️ NIET veranderen

3️⃣ Je expo-updates plugin is OK

Dit is ook goed:

[
  "expo-updates",
  {
    "username": "glow-up-sports"
  }
]


Geen issues hier 👍

4️⃣ Je eas.json is GOED (geen fout)

Dit is netjes opgezet:

preview → channel: preview

production → channel: production

env vars correct

👉 Hier hoef je niks aan te passen

⚠️ MAAR… 1 KEER opnieuw builden is noodzakelijk

Omdat:

OTA-config in app.json wordt ingebakken in de APK

Dus je moet NU doen:
eas build -p android --profile preview


👉 installeer deze nieuwe APK
👉 daarna werkt OTA 100%

Daarna is je flow simpel (en eindelijk chill 😌)
Voor preview testers:
eas update --branch preview --message "UI tweak"

Voor live users:
eas update --branch production --message "Bugfix login"


🚀 Geen nieuwe APK meer nodig
🚀 Exact hoe grote apps het doen

🧠 Waarom dit zo verwarrend is (even geruststelling)

Expo:

toont downloads

toont update groups

toont assets

maar zegt NIET “ik gebruik embedded JS”

Dus het voelt alsof OTA “half werkt”
Maar dit ene veld bepaalt alles 👇

"fallbackToCacheTimeout": 0
