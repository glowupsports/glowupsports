1️⃣ Rainbow test banner verwijderen

Ja, correct.
Alles wat test/preview/experiment is → weg uit production.

👉 Dit voorkomt:

review-vragen

“test content zichtbaar in production”

verwarring bij testers

2️⃣ Production build type → AAB (niet APK)

100% correct.
Play Store accepteert geen APK’s meer (behalve legacy).

Check in eas.json:

{
  "build": {
    "production": {
      "android": {
        "buildType": "app-bundle"
      }
    }
  }
}


✔️ Dit is verplicht voor Play Store.

3️⃣ versionCode in app.json – hier zit nuance

👉 Niet strikt verplicht als:

autoIncrement: true aan staat

je al minstens één build hebt gedaan

❗ Maar best practice = WEL toevoegen als startpunt.

Aanrader:

{
  "expo": {
    "android": {
      "package": "com.glowupsports.app",
      "versionCode": 1
    }
  }
}


Daarna:

EAS → 2, 3, 4, 5…

jij hoeft nooit meer te denken

✔️ Dus: je plan is correct, dit is slim.

4️⃣ Expo Updates uitschakelen voor production

Ja — en dit is een HEEL goede call na je OTA-ellende.

Je hebt twee veilige opties:

🔒 Optie A — Updates UIT in production (meest stabiel)
{
  "expo": {
    "updates": {
      "enabled": false
    }
  }
}


✔️ Play Store updates only
✔️ Geen OTA download errors
✔️ Review-proof

🧠 Optie B — Updates alleen voor preview

(kan later, nu niet nodig)
