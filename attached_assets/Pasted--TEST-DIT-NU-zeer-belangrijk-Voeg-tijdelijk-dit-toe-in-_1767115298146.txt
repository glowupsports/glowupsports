🔧 TEST DIT NU (zeer belangrijk)

Voeg tijdelijk dit toe in je app (bijv. in App.tsx):

import * as Updates from 'expo-updates';

if (!__DEV__) {
  Updates.reloadAsync();
}


➡️ Build opnieuw
➡️ Open app
➡️ Als rainbow weg is → dit was het probleem

👉 Oplossing daarna: reload alleen bij updateAvailable.

2️⃣ Android cache houdt oude bundle vast (ZEER vaak)

Android is hier berucht om.

🔧 DOE DIT OP JE TOESTEL:

Instellingen → Apps → Glow Up Sports

Opslag

Clear cache (NIET clear data eerst)

App volledig sluiten

App openen

⚠️ Alleen “swipe weg” is vaak niet genoeg op Android.

3️⃣ Je rainbow zit in een native-achtige plek

OTA kan GEEN native code of config aanpassen.

Als de rainbow:

in app.json zit

in splash / status bar / layout config

of via een native lib

❌ Dan zal OTA nooit werken.

Check dit:

Is de rainbow:

❓ een React component → OTA kan

❌ via StatusBar, NavigationContainer theme, app.json → nieuwe APK nodig

4️⃣ De code wordt nooit “geraakt”

Dit gebeurt vaker dan mensen denken.

Bijv:

const SHOW_RAINBOW = true;


Of:

if (process.env.EXPO_PUBLIC_FLAG) { ... }


Maar:

env vars zijn ingebakken in build

OTA kan env vars niet veranderen

➡️ Dus codepad blijft hetzelfde.

5️⃣ Je gebruikt (onbewust) een Development Client

Als de app:

“Dev menu” heeft

shake → dev tools toont

Dan geldt:

❌ OTA werkt ANDERS / soms niet

Check:

Is dit een internal distribution build zonder dev client?

In eas.json:

"preview": {
  "distribution": "internal",
  "developmentClient": false
}


Als developmentClient: true → dit verklaart alles

🧠 Mijn professionele conclusie (op basis van alles)

Er zijn maar 3 realistische uitkomsten:

🔁 App reloadt niet → fix met Updates.reloadAsync

🧠 Rainbow zit in native/config → nieuwe APK nodig

🧪 Dev client / env var → OTA kan ‘m niet aanpassen

Het is géén channel, branch of EAS probleem meer.
