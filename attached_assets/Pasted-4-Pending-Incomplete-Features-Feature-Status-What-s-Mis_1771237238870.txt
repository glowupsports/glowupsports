6. Bugs and Errors
Critical Issues
CORS Header Missing X-Active-Player-Id: The CORS setup in server/index.ts allows headers "Content-Type, Authorization, Cache-Control, X-Academy-Id" but does NOT include X-Active-Player-Id - this means family player switching will fail with CORS errors on web and potentially on native pre-flight requests.

routes.ts is 37,474 lines: This is a single file with 630+ endpoints. It's a maintainability time bomb - any edit risks breaking unrelated features, it's slow to parse/lint, and impossible to code review.

No Account Deletion in Player App: Google Play Store requires apps to allow users to delete their accounts. The server has a /api/delete-account-request endpoint but the player app has no UI to trigger it.

No Privacy Policy / Terms of Service in Player App: The coach settings screen links to glowupsports.com/privacy and glowupsports.com/terms, but the player settings screen has neither.

Security Concerns
No Rate Limiting: All 630+ endpoints have zero rate limiting. Login attempts, API calls, account creation - all unlimited. This makes the app vulnerable to brute force attacks and API abuse.

Minimal Input Validation: Only ~6 endpoints use Zod validation out of 630+. Most endpoints directly trust req.body data without sanitization.

JWT Secret Handling: The JWT signing works but should be verified that JWT_SECRET is a strong, environment-specific value.

Performance Concerns
No Pagination on Most Lists: With 71 screens but only 4 pagination patterns, most screens load all data at once. As the platform grows, this will cause performance degradation.

CommunityScreen.tsx is 5,351 lines: This single component is extremely large and likely has render performance issues.

Earnings calculation takes 3+ seconds: Server logs show [Earnings PERF] Parallel calculation done: 3182 ms - this is cached but cold-start users will see significant delay.

UI/UX Issues
"Unexpected text node" errors: Browser console shows React Native text rendering errors that indicate stray characters in JSX somewhere.

Only 1 accessibility label: The player app is essentially inaccessible to screen reader users.
