🎯 HET GROTE VRAAGSTUK

Hoe ziet vooruitgang + feedback eruit voor een coach, zónder dat het zwaar of traag wordt?

De sleutel is:
👉 Scheiding tussen 3 dingen

Lesvoorbereiding

Les-evaluatie (direct na les)

Langetermijn voortgang (overzicht)

Als je die mengt → chaos.
Als je ze scheidt → rust + kwaliteit.

🧠 HOOFDSTRUCTUUR (COACH APP)

Ik stel 1 apart hoofdgedeelte voor in de Coach App:

🧩 “COACHING” of “PLAYER PROGRESS”

(naast Kalender in de bottom / side nav)

Binnen dit gedeelte heb je 3 tabs:

1️⃣ Today → Feedback
2️⃣ Players → Progress
3️⃣ Plans → Preparation

Ik leg ze één voor één uit.

1️⃣ TODAY → FEEDBACK
(dit is je dagelijkse coach-werkplek)

“Ik heb vandaag 5 lessen gehad. Nu wil ik alles afronden in 10–15 minuten.”

Wat de coach ziet

Automatische lijst:

Alle lessen van vandaag

Gesorteerd op tijd

Per les:

Les type

Groep / spelers

Status:

⏳ Feedback open

✅ Afgerond

➡️ Dit voelt als een to-do lijst, geen rapportage.

Binnen één les (Feedback Flow)
Stap 1 — Groepsniveau (optioneel, snel)

Session intensity:

🟢 Light

🟠 Normal

🔴 Intense

Forehand

⏱️ 20–30 seconden.

Stap 2 — Spelers beoordelen (supersimpel)

A. Attendance (al ingevuld vanuit kalender)

B. Performance quick rating (NO NUMBERS)

⬇️ Needs work

(1 tap)

🔥 High

🙂 Normal

⚠️ Low

D. Optionele coach note

max 140 tekens

volledig optioneel

➡️ Per speler: 5–10 seconden.

Stap 3 — Save & Done

“Save feedback”

Les wordt:

vergrendeld

verwerkt in progress

zichtbaar voor Player App (later)

Waarom dit werkt

Geen cijfers

Geen technische overload

3️⃣ PLANS → PREPARATION
(voor coaches die wíllen voorbereiden — niet verplicht)

Dit is optioneel, maar goud voor kwaliteit.

Lesvoorbereiding – hoe simpel moet het zijn?
Wat we NIET doen

❌ Geen volledige lesplannen

❌ Geen oefeningen-database (nu)

❌ Geen verplichte formats

Wat we WEL doen (coach-vriendelijk)
A. Session Template (light)

Coach kan opslaan:

Les type

Ball level

Skill level

Focus areas (tags)

Verwachte intensiteit

B. Koppelen aan kalender

“Use preparation template?”

achteraf koppelen
