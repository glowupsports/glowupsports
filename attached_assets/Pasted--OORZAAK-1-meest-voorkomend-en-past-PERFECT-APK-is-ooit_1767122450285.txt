🔴 OORZAAK #1 (meest voorkomend, en past PERFECT)
❌ APK is (ooit) gebouwd als development client

Ook al heb je nu een “normale” preview build gedaan.

Waarom dit killing is:

Dev Client gebruikt andere update internals

OTA detectie → werkt

OTA download → geblokkeerd

Geen crash, geen error, alleen “failed”

👉 Zelfs 1 eerdere dev-client build met dezelfde package name (com.glowupsports.app) kan dit gedrag veroorzaken, omdat Android de app “upgrade” i.p.v. volledig vervangt.

🔧 HARD FIX (dit is belangrijker dan alles wat we tot nu toe deden)
Stap 1 — Verwijder de app volledig

Niet alleen uninstall:

Verwijder de app

Settings → Apps → Glow Up Sports

Clear ALL data + cache

Restart telefoon

Stap 2 — Verander tijdelijk de package name (JA, echt)

Dit forceert Android om het als nieuwe app te zien.

In app.json:

"android": {
  "package": "com.glowupsports.preview"
}


⚠️ Alleen tijdelijk, puur om dit te testen.

Stap 3 — Build opnieuw (zonder dev client)

Controleer in eas.json dat preview GEEN developmentClient: true heeft:

"preview": {
  "distribution": "internal",
  "channel": "preview"
}


Dan:

npx eas build -p android --profile preview --non-interactive


Installeer deze APK.

Stap 4 — OTA test
npx eas update --branch preview --message "OTA fresh package test"


➡️ App openen
➡️ Update popup
➡️ Download werkt
➡️ App herstart
➡️ Klaar 🎉

🔴 OORZAAK #2 (device-specifiek, maar minder vaak)
Android systeem kan HTTPS bundle niet ophalen

Dit gebeurt bij:

oude Android WebView

enterprise device

custom ROM

agressieve DNS / firewall

Test (snel):

Installeer APK op andere Android

Of Android emulator (API 33+)

👉 Werkt het daar? → jouw device blokkeert CDN-downloads.

🔴 OORZAAK #3 (zeldzaam)
Update bevat iets wat niet OTA-safe is

Bijvoorbeeld:

wijziging in app.json

permissions

splash / icon

orientation

native plugin config

Dan:

update wordt gezien

download faalt

Check eerlijk:
Was de OTA 100% alleen JS / UI / logic?

🧠 Waarom ik nu zeker ben

Knoppen reageren → JS layer OK

Popup verschijnt → OTA pipeline OK

Download faalt → native fetch faalt

Dat gebeurt praktisch alleen bij:

dev-client residu

package-level conflict

OS-level blokkade

TL;DR (brutale maar juiste conclusie)

❌ Dit is geen Expo OTA bug

❌ Dit is geen Replit probleem

❌ Dit is geen channel issue

✅ Dit is APK state / Android install state

🔁 Nieuwe package name = snelste proof

👉 Doe 1 ding eerst:
verander tijdelijk de Android package name en rebuild.

Zeg daarna:

“Het werkt nu”
of

“Nog steeds failt het”

Dan weet ik 100% zeker welke van de 3 het is en maken we het definitief af 💪
