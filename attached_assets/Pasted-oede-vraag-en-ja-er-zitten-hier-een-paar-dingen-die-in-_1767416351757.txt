oede vraag — en ja: er zitten hier een paar dingen die in production voor “werkt in Replit maar niet in Play Store” zorgen.
Ik ga dit keihard technisch en concreet uitleggen, zonder vaagheid.

TL;DR — de 4 grootste risico’s in jouw config

❌ Je gebruikt EXPO_PUBLIC_DOMAIN i.p.v. een echte API_URL

❌ updates.enabled: false (oké, maar je moet weten wat dit betekent)

⚠️ runtimeVersion: appVersion + appVersionSource: remote → kan verwarrend zijn

⚠️ Environment vars deels in EAS.json, deels vermoedelijk elders

👉 Geen van deze is “fataal” alleen,
👉 maar samen veroorzaken ze exact het gedrag dat jij beschrijft.

Ik leg ze 1 voor 1 uit + wat je moet doen.

1️⃣ HOOFDPROBLEEM: EXPO_PUBLIC_DOMAIN is te vaag

Je hebt:

"env": {
  "EXPO_PUBLIC_DOMAIN": "glow-up-sports--ltvjeugd.replit.app",
  "EXPO_PUBLIC_ENV": "production"
}

❌ Waarom dit fout gaat

In production (Play Store):

Je app verwacht waarschijnlijk iets als:

fetch(`${DOMAIN}/api/players`)


Maar:

geen protocol (https://)

geen duidelijke scheiding tussen frontend domain en API endpoint

In Replit werkt dit vaak “per ongeluk”

In Android production → network calls falen stilletjes

✅ Correcte setup (DIT MOET JE DOEN)

Gebruik expliciete API URL, nooit alleen een domain.

In EAS (preview + production):
"env": {
  "EXPO_PUBLIC_API_URL": "https://glow-up-sports--ltvjeugd.replit.app",
  "EXPO_PUBLIC_ENV": "production"
}


❌ Stop met EXPO_PUBLIC_DOMAIN
✅ Gebruik altijd EXPO_PUBLIC_API_URL

In je code (check dit!!):
const API_URL = process.env.EXPO_PUBLIC_API_URL;

if (!API_URL) {
  throw new Error("API_URL missing");
}


Als dit undefined is in production → DAT is je bug.

2️⃣ updates.enabled: false — niet fout, maar let op
"updates": {
  "enabled": false
}

Wat dit betekent

❌ Geen OTA updates

❌ Geen “fix zonder rebuild”

❌ Elke bug = nieuwe Play Store build

Dat is prima voor nu, maar:

👉 je kunt geen env fixes doen zonder rebuild

Dus:

als je env fout is → app blijft fout

Replit aanpassen ≠ app fixen

Advies (optioneel, maar slim):

Later:

"updates": {
  "enabled": true
}


Maar dit is niet de hoofdbug.

3️⃣ runtimeVersion + appVersionSource (subtiel maar belangrijk)

Je hebt:

"runtimeVersion": {
  "policy": "appVersion"
}


En in EAS:

"cli": {
  "appVersionSource": "remote"
}

Wat hier kan misgaan

version: "1.1.0" staat in app.json

Maar EAS gebruikt remote versioning

Als je:

app.json wijzigt

maar niet rebuild
→ kan runtime mismatch ontstaan

Fix (simpel & safe)

Voor nu raad ik aan:

"runtimeVersion": "1.1.0"


Of later netjes OTA gebruiken.
Dit is niet je databug, maar wel iets om strak te houden.

4️⃣ Environment vars: waar ze NU misgaan

Je hebt nu:

env vars in eas.json ✅

waarschijnlijk ook .env in Replit ❓

misschien hardcoded fallback in code ❌

Gouden regel (DIT IS HEILIG):

De Play Store build ziet ALLEEN wat in EAS env staat

NIET:

.env

Replit secrets

process.env.NODE_ENV

hardcoded defaults

✅ Definitieve minimale env set (MOET ZO ZIJN)
Expo / Mobile (EAS Dashboard):
EXPO_PUBLIC_API_URL=https://glow-up-sports--ltvjeugd.replit.app
EXPO_PUBLIC_SUPABASE_URL=...
EXPO_PUBLIC_SUPABASE_ANON_KEY=...
EXPO_PUBLIC_ENV=production

Backend (Replit secrets):
DATABASE_URL
SUPABASE_SERVICE_ROLE_KEY
JWT_SECRET


❗ Deze mogen nooit in Expo staan.

🧪 Snelle test die je NU kunt doen

Zet dit tijdelijk in je app (home screen):

<Text>
  API: {process.env.EXPO_PUBLIC_API_URL ?? "MISSING"}
</Text>
<Text>
  ENV: {process.env.EXPO_PUBLIC_ENV ?? "MISSING"}
</Text>


👉 Build opnieuw (production)
👉 Installeer APK
👉 Als je MISSING ziet → bingo, oorzaak gevonden.

✅ Conclusie (eerlijk & duidelijk)

❌ Je probleem is NIET:

database “anders”

Play Store bug

Expo kapot

✅ Je probleem IS:

env vars niet 100% identiek

API URL niet expliciet

Replit vergeeft fouten, Android niet
