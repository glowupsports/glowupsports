7) Replit / EAS iOS build — dit is waarom jouw commands faalden

Je errors (“could not determine executable”, “eas: command not found”) = EAS CLI staat niet goed in je environment / je riep eas aan zonder dat hij bestaat.

✅ Exacte Replit Shell commands (copy-paste)

Doe dit in Replit Shell:

# 1) check node
node -v

# 2) installeer EAS CLI in je project
npm i -D eas-cli

# 3) check of EAS werkt
npx eas-cli --version

# 4) login
npx eas-cli login

Daarna je iOS build:

Voor TestFlight / App Store build (wat jij nodig hebt)
npx eas-cli build --platform ios --profile production

Optioneel auto-upload naar App Store Connect:

npx eas-cli build --platform ios --profile production --auto-submit

Jij gebruikte npx eas-cli build --platform android --profile production voor Android. Voor iOS is het dus exact hetzelfde, alleen ios.

8) Je eas.json mist iOS settings (aanrader)

Je hebt bij preview en production wel Android config, maar geen iOS autoIncrement.

Voeg toe (minimaal):

"preview": {
  "distribution": "internal",
  "channel": "preview",
  "ios": { "autoIncrement": true },
  "android": { "autoIncrement": true },
  "env": { ... }
},
"production": {
  "channel": "production",
  "ios": { "autoIncrement": true },
  "android": { "buildType": "app-bundle", "autoIncrement": true },
  "env": { ... }
}

Waarom: iOS build numbers moeten omhoog bij nieuwe uploads.
