🧠 Eerst: wat bedoelt Expo met “version control”?

Expo bedoelt hier Git.

Niet:

database versioning

app versions

OTA versions

Maar:

Git = geschiedenis van je code
(zodat OTA updates altijd exact weten wat ze deployen)

🚀 STAP-BY-STAP: VERSION CONTROL OPZETTEN (Git)
1️⃣ Check of je al in een git repo zit

In je project root:

git status

Mogelijke uitkomsten:

❌ fatal: not a git repository → ga door naar stap 2

✅ toont files → je zit al in git (dan slaan we init over)

2️⃣ Git initialiseren (1x)
git init


Je ziet:

Initialized empty Git repository


✔️ Klaar.

3️⃣ Zorg dat je .gitignore goed is (BELANGRIJK)

Maak of check .gitignore in dezelfde map als app.json.

Gebruik minimaal dit:
# Node
node_modules
npm-debug.log
yarn-error.log

# Expo
.expo
.expo-shared
web-build

# Env
.env
.env.*

# Replit
.replit
.replit.nix
.cache

# OS / Editor
.DS_Store
.vscode
.idea


👉 Dit voorkomt dat Expo / Replit rommel commit.

4️⃣ Eerste commit maken
git add .
git commit -m "Initial Glow Up Sports setup"


🎉 Dit is je ankerpunt
Vanaf hier is alles veilig.

5️⃣ (OPTIONEEL MAAR AANRADER) GitHub koppelen

Dit hoef je niet NU te doen, maar is slim.

a) Maak lege repo op GitHub (geen README)
b) Koppel:
git remote add origin https://github.com/JOUWNAAM/glow-up-sports.git
git branch -M main
git push -u origin main


Nu heb je:

lokale backup

cloud backup

teamwork-ready setup

🔥 Wat dit direct oplost voor EAS / OTA

Na Git:

Je kunt OTA gewoon doen met:
CI=1 npx eas update --branch preview --message "Rainbow banner removed"


En Expo:

❌ stopt met die waarschuwing

✅ koppelt OTA updates aan commits

✅ geeft betere traceability in dashboard

🧠 Hoe jij dit het beste gebruikt (Glow Up workflow)
Mijn advies voor jou:

Elke logische change → commit

OTA updates mogen meerdere commits bevatten

Bijvoorbeeld:

git commit -m "Remove rainbow banner from header"
git commit -m "Improve dashboard spacing"
CI=1 npx eas update --branch preview --message "Dashboard UI cleanup"


✔️ Code geschiedenis blijft netjes
✔️ OTA blijft clean

🧱 Simpele regels (onthoud deze)

❌ NOOIT node_modules committen

❌ NOOIT .env committen

✅ WEL commits vóór belangrijke OTA’s

✅ WEL duidelijke commit messages

🟢 Samenvatting

Expo waarschuwde terecht

Git is de oplossing

Setup kost 5 minuten
