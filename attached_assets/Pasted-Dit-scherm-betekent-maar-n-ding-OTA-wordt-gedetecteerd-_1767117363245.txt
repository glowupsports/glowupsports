Dit scherm betekent maar één ding:

✅ OTA wordt gedetecteerd
❌ Download faalt op netwerk/config-niveau

Dus: je OTA-pipeline leeft. Nu fixen we alleen nog waarom de download faalt.

Wat dit scherm technisch betekent

Expo ziet een nieuwere update

App vraagt toestemming om te updaten

Download start, maar wordt afgebroken

Dat is geen build-fout, maar één van de onderstaande oorzaken 👇

🔴 Meest waarschijnlijke oorzaken (in jouw setup)
1️⃣ Network / SSL blokkade (nr. 1 oorzaak)

Dit gebeurt vaak bij:

bedrijfs-wifi

VPN

DNS filtering

private Android network settings

Waarom: OTA assets komen van https://u.expo.dev + CDN.

Test (belangrijk):

Zet wifi uit

Gebruik 4G/5G

Druk op Try Again

👉 Als dit werkt → het is netwerk-blocking, geen code.

2️⃣ App heeft geen internet permission (Android)

Check in app.json of dit bestaat:

"android": {
  "permissions": ["INTERNET"]
}


Zonder dit:

update wordt gevonden

download faalt

⚠️ Soms ontbreekt dit bij custom configs.

3️⃣ Oude preview APK met oude OTA-config

Als je:

vóór de updates fix een APK had

die niet volledig verwijderd is

Dan kan:

OTA URL fout zijn

fallback gedrag verkeerd zijn

Fix (snel):

Verwijder app volledig

Installeer nieuwste preview APK

Probeer opnieuw

4️⃣ CDN timing / propagation (komt voor bij snelle updates)

Als je:

net een eas update hebt gedaan

en direct update probeert

Dan kan CDN nog niet “warm” zijn.

Fix:

Wacht 1–2 minuten

Druk op Try Again

5️⃣ Update bevat iets wat niet OTA-safe is

OTA kan geen native changes doen.

Voorbeelden:

nieuwe permissions

native modules

splash / icon / orientation changes

Android config changes

Dan:

update wordt aangeboden

download faalt / wordt geweigerd

Check:
Was je update alleen JS/UI/logic?
→ Dan is dit niet de oorzaak.
Was het config/native?
→ Dan is een nieuwe build nodig.

🔍 Hoe jij dit NU het snelst debugt (2 minuten)
Stap 1 — Force retry

Zet wifi uit

Open app

Klik Try Again

Stap 2 — Log check (optioneel maar goud)

Run:

adb logcat | grep expo


Je ziet dan exact:

SSL error

timeout

permission error

Waarom dit goed nieuws is (belangrijk)

Dit is het laatste stadium vóór succes:

❌ vóór: “OTA werkt niet”
✅ nu: “OTA werkt, download faalt”

Dat is altijd fixbaar zonder architectuurwijziging.
