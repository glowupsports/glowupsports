✅ STAP 2 — Player Home Screen
“Coach doet → Player ziet → Systeem onthoudt”

Dit scherm is het hart van de Player App.
Alles wat de coach doet, resulteert hier. Geen losse features meer.

1️⃣ Rol van de Player Home (heel belangrijk)

De Player Home is GEEN menu
De Player Home is GEEN dashboard vol grafieken

👉 Het is een dagelijkse briefing.

De speler opent de app en ziet meteen:

Waar sta ik?

Wat heb ik gedaan?

Wat komt eraan?

Wat verwacht mijn coach van mij?

2️⃣ Structuur van de Player Home (van boven naar beneden)
🔹 A. Player Header (compact, calm)

Doel: identiteit + status, niet schreeuwerig

Inhoud:

Avatar (klein)

Naam

Level

XP bar (subtiel)

Glow Score (tap → uitleg)

Alex • Level 7
[██████████░░░] 2450 / 3000 XP
Glow Score: 847


❌ Geen coins hier
❌ Geen achievements hier

🔹 B. “Coach Summary Card” (BELANGRIJKSTE BLOCK)

Dit is de magie.

“Wat heeft mijn coach vandaag geregistreerd?”

Voorbeelden:

“Coach Maria heeft feedback toegevoegd”

“Je forehand consistency is verbeterd”

“1 sessie vandaag • intensity: high”

“Coach heeft een focuspunt gezet”

UI:

1 kaart

Groen/blauw accent

Coach naam zichtbaar

Coach Feedback
✔ Session completed
✔ Focus: Court Positioning
✔ Intensity: 8/10


👉 Alles hier komt DIRECT uit coach actions
👉 Geen handmatige player input

🔹 C. Today Snapshot (max 3 metrics)

Geen overload.

Altijd exact deze 3:

XP today

Sessions today / this week

Mood / Energy (coach input)

Today
+120 XP
1 Session
Energy: Good

🔹 D. Next Session (calendar sync)

Niet optioneel.

Next Session
Today • 18:00
Court 3 • Coach Maria


CTA:

“View Session”

“Mark Attendance” (optioneel later)

🔹 E. System Message (collapsed)

Je had dit al — dit is goed, maar:

Gebruik het spaarzaam.

Voorbeelden:

“Jake leveled up”

“New feedback available”

“Coach updated your focus”

Niet elke XP trigger.

3️⃣ Wat hier ABSOLUUT niet hoort (bewust!)

❌ Quest lijst
❌ Friends
❌ Rankings
❌ Matches
❌ Shop
❌ Events

Die horen in hun eigen tabs.

Player Home = dagelijkse focus

4️⃣ Data-flow (dit is cruciaal)
Coach → API → Player Home

Voorbeeld:

Coach doet:

Attendance markeren

Feedback invullen

Intensity kiezen

Focus area selecteren

API:

POST /sessions/{id}/complete


Backend:

Update progress tables

Log XP transaction

Update player status

Player Home:

GET /player/home


En krijgt:

{
  "coachSummary": {...},
  "todayXp": 120,
  "nextSession": {...}
}


👉 Player Home doet geen berekeningen
👉 Alleen presenteren

5️⃣ UX-principes (waar je huidige design misgaat)

Je huidige mockups zijn visueel cool, maar:

Te veel info tegelijk

Te game-achtig voor dagelijks gebruik

Geen duidelijke hiërarchie

Stap 2 draait om:

Rust

Vertrouwen

Coach-authoriteit

Gamification komt later en gedoseerd.

6️⃣ Definition of Done — Stap 2

Je bent klaar met Stap 2 als:

✅ Player Home opent instant
✅ Coach acties verschijnen zichtbaar
✅ Geen dummy data
✅ Geen overload
✅ Player snapt: “ik hoef niks te doen, ik volg”
