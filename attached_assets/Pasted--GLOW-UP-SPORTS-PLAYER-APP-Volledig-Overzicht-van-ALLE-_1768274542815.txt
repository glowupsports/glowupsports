🎾 GLOW UP SPORTS – PLAYER APP
Volledig Overzicht van ALLE Openstaande Issues, Gaten & Risico’s

Dit is alles wat nog niet af is, half af is, fout kan gaan of launch blokkeert.

🔴 A. KRITIEKE PROBLEMEN (LAUNCH BLOCKERS)

Deze dingen moeten opgelost zijn vóór publieke launch.

1. ❌ Payment Processing (Stripe)

Status: Niet geïmplementeerd
Impact: Monetisatie onmogelijk

Problemen:

Geen Stripe checkout

Geen betaling voor:

Credits

Shop producten

Services

Court bookings

Marketplace transacties zijn “fake” (browse-only)

Betrokken screens:

ShopScreen

MarketplaceScreen

ParentCreditStore

CourtBookingScreen

LessonBookingScreen

👉 Zonder dit is de app alleen een demo

2. ❌ Real-time Chat (WebSocket)

Status: UI bestaat, geen echte communicatie
Impact: Onbruikbaar voor coaches & spelers

Problemen:

Geen WebSocket verbinding

Geen live updates

Geen:

Typing indicators

Read receipts

Push bij nieuw bericht

Messages zijn statisch / verouderd

API’s niet gekoppeld:

/api/player/messages

/api/player/messages/send

Betrokken screens:

PlayerMessagesScreen

ChatInbox (coach-side)

3. ❌ Push Notifications (Triggers ontbreken)

Status: Token registratie bestaat, maar triggers niet
Impact: Gebruikers missen cruciale events

Wat NIET werkt:

Geen push bij:

Nieuwe sessie

Sessie wijziging

Chatbericht

Feedback

Level up

Alleen lokale notificaties / UI lijst

API deels aanwezig maar niet gebruikt

4. ❌ Video Playback voor Skill Evidence

Status: Opnemen werkt, bekijken niet
Impact: Vertrouwen kapot

Problemen:

Geen video player

Evidence kan niet:

Herbekeken worden

Gedeeld worden

Door coach/parent bekeken worden

API ontbreekt / niet gekoppeld:

/api/player/evidence/:id/video

🟠 B. GROTE FUNCTIONELE GATEN (HIGH PRIORITY)
5. ⚠️ Lesson Booking = Stub

Status: Bestaat amper (32 regels code)

Wat ontbreekt:

Coach selectie

Tijdslot selectie

Type les

Credit check & deductie

Bevestiging & betaling

👉 Dit is zichtbaar niet af en moet óf af, óf verborgen worden.

6. ⚠️ Court Booking Half Af

Status: UI ok, logica fake

Problemen:

Time slots zijn hardcoded

Prijzen zijn hardcoded

Geen betaling

Geen conflict check met echte data

7. ⚠️ Player Notifications Screen = UI Only

Problemen:

“Mark as read” werkt niet betrouwbaar

Geen sync met backend

Geen koppeling met push systeem

🟡 C. UX / PRODUCT PROBLEMEN (MID PRIORITY)
8. ⚠️ Nieuwe Player (Level 1) = Verwarrend

Probleem:

Bijna alles is locked

Geen duidelijke “wat nu?” flow

Home voelt leeg / frustrerend

Ontbreekt:

Guided onboarding na signup

“Next steps” card

Safe unlocked actions

9. ⚠️ Geen Empty States

Probleem:

Lege schermen zonder uitleg

Betroffen o.a.:

Match history

Friends list

Messages

Community feed

Marketplace (geen listings)

10. ⚠️ Geen Skeleton Loaders

Probleem:

Abrupte loading

Slechte perceived performance

Alleen ActivityIndicator gebruikt.

11. ⚠️ Geen Pagination

Probleem:

Performance issues bij:

Community feed

Leaderboard

Friends list

Alles wordt in één keer geladen.

12. ⚠️ Hardcoded Currency

Probleem:

Overal “AED”

Negeert academy currency

Betroffen:

Shop

Marketplace

Credits

Parent portal

🟡 D. TECHNISCHE RISICO’S (MID PRIORITY)
13. ⚠️ Extreem Grote Files

Probleem:

Slecht onderhoudbaar

Kans op bugs bij wijzigingen

Top offenders:

PlayerHomeScreen – 3.905 regels

PlayerProfileScreen – 1.924 regels

CommunityScreen – 1.742 regels

👉 Niet direct kapot, maar gevaarlijk op termijn

14. ⚠️ Geen Error Boundaries

Probleem:

Eén crash = hele app dood

Geen fallback UI

15. ⚠️ Query Over-Fetching

Probleem:

Meerdere API calls voor dezelfde data

Geen batching / memoization

Impact:

Slechtere performance

Onnodig dataverbruik

16. ⚠️ Geen Offline Mode

Probleem:

Zonder internet = app onbruikbaar

Geen cache fallback
