#23 - Fix Inflated Debt Count (Cancelled Sessions)
What & Why
Players like Julia Waheb are shown a debt that is 1 (or more) higher than their actual attended session count. She has 13 visible "Present" sessions but the app shows -14 group and "14 group session(s) attended without active package." The root cause: when a coach marks a player Present and then later cancels the entire session, the debt transaction that was created at mark-time is NOT cancelled — it stays unsettled. Because that session no longer appears in the player's visible history (cancelled sessions are hidden), the player is billed for a ghost session. The same logic flaw can also inflate counts when cancelled sessions are included in the "missingSessions" fallback inside ensureCreditProcessed.

Done looks like
Julia's group debt shows -13 (matching her 13 real attended sessions), not -14.
Any player who had a session marked Present and then the session was cancelled by the coach does NOT carry debt for that cancelled session.
The "N session(s) attended without active package" warning counts only sessions that actually took place (not cancelled).
The home screen "-1 Grp / Awaiting Payment" badge also corrects itself once the ghost debt is removed.
A backfill/repair step cancels existing ghost debt transactions in the database so current users are fixed immediately (not just future sessions).
Out of scope
Changing how debt is settled when a player buys a new package.
UI changes to the PackagesCard beyond the number fixing itself.
Tasks
Cancel debt on session cancellation — In the session-cancellation code path (wherever a session's status is set to cancelled/deleted), call cancelSessionDebt for every player who had an unsettled debt transaction linked to that session. This prevents future ghost debts.
Fix ensureCreditProcessed / attendance change — When a player's attendance is changed away from "present"/"late" (to absent, vacation, or the session itself is cancelled), ensure any existing unsettled debt for that session+player is marked cancelled: true in metadata, so it is excluded from getPlayerCreditBalanceByType.
One-time backfill — Add a repair routine (can run as part of the existing player repair endpoint or a migration) that finds all credit_transactions where isDebt=true, settled=false, cancelled=false, and the linked session has status cancelled or deleted — and marks those transactions cancelled: true. This fixes existing affected players like Julia without requiring manual intervention.
