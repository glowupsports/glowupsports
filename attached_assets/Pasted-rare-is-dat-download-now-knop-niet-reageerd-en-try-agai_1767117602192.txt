Ja — dit gedrag is heel kenmerkend, en eigenlijk nóg beter nieuws dan je denkt 😄
Dit vertelt ons precies waar het vastloopt.

Wat dit gedrag betekent (belangrijk)

❌ Download Now reageert niet
❌ Try Again reageert niet
✅ Terug / Later werkt wel

👉 Dit betekent NIET:

geen netwerk

geen CDN

geen server issue

👉 Dit betekent WEL:

Expo Updates UI wordt getoond, maar de download-handler crasht / wordt geblokkeerd vóór hij start

Met andere woorden:
de update-flow wordt afgebroken op native-config niveau, nog vóór een request wordt gedaan.

🔥 DE ECHTE ROOT CAUSE (zeer duidelijk nu)
❌ expo-updates zit er WEL in
❌ maar de Android native permissions / config zijn niet correct ingebakken in de APK

Daardoor:

modal opent

buttons renderen

maar onPress → native download() faalt stilletjes

geen error → geen retry → knop lijkt “dood”

Dit is klassiek bij Expo + Android als één van deze dingen ontbreekt.

✅ DE 3 CONCRETE OORZAKEN (in jouw setup)
1️⃣ INTERNET permission ontbreekt in de APK

Ook al “heeft elke app internet” — Expo Updates vereist expliciet INTERNET.

👉 Fix in app.json:

"android": {
  "permissions": ["INTERNET"]
}


⚠️ Dit vereist een nieuwe APK build
OTA kan dit NIET fixen.

2️⃣ Oude preview APK met verkeerde Updates-config

Je hebt eerder:

APK gebouwd vóórdat we dit toevoegden:

"checkAutomatically": "ON_LOAD",
"fallbackToCacheTimeout": 0


Gevolg:

Update UI werkt

Download mechanisme is inconsistent

Buttons doen niks

👉 Oude APK = per definitie onbetrouwbaar

3️⃣ Android 13+ notification / background restrictions (secundair)

Expo Updates download gebeurt soms via background task.

Zonder correcte setup:

tap → geen actie

geen crash

geen log

Maar dit wordt automatisch opgelost zodra INTERNET correct zit.

🧨 Waarom er GEEN foutmelding is

Expo Updates:

faalt stil

Android slikt de call

UI blijft staan

Dit is een bekende frustratie in Expo (niet jouw fout).

✅ DE DEFINITIEVE FIX (geen omwegen)
🔧 Stap 1 — Pas app.json aan (DIT IS VERPLICHT)

Zorg dat je android block er zo uitziet:

"android": {
  "permissions": ["INTERNET"],
  "adaptiveIcon": {
    "backgroundColor": "#1A1A1A",
    "foregroundImage": "./assets/images/android-icon-foreground.png",
    "backgroundImage": "./assets/images/android-icon-background.png",
    "monochromeImage": "./assets/images/android-icon-monochrome.png"
  },
  "edgeToEdgeEnabled": true,
  "predictiveBackGestureEnabled": false,
  "package": "com.glowupsports.app"
}

🔧 Stap 2 — VOLLEDIG nieuwe preview APK bouwen
npx eas build -p android --profile preview --non-interactive

🔧 Stap 3 — Oude app verwijderen

⚠️ Niet updaten, echt verwijderen:

App verwijderen

Cache weg

Nieuwe APK installeren

🔧 Stap 4 — OTA opnieuw testen
npx eas update --branch preview --message "OTA final test"


➡️ App openen
➡️ Update popup
➡️ Download Now werkt
➡️ App herstart
➡️ Nieuwe code zichtbaar 🎉
