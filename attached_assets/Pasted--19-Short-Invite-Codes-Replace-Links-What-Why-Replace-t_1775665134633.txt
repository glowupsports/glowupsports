#19 - Short Invite Codes Replace Links
What & Why
Replace the current 16-char hex invite codes (shared as long URLs) with short 6-character uppercase alphanumeric codes (e.g., GW4K9P). Short codes are safer to share verbally/by message, look more professional, and avoid exposing the server domain in shared links.

Done looks like
Creating a player invite shows a clean 6-char code (e.g., GW4K9P) instead of a URL
The code is displayed large and prominently with a "Copy Code" button (no URL shown)
The join/onboarding screen says "Enter your invite code" — simple text field, no mention of "paste a link"
Entering the 6-char code on the join screen still validates and works correctly
Academy-level invites (Admin invite links screen) also show short codes instead of URLs
Out of scope
Changing the invite flow itself (creation, claiming, expiry logic stays the same)
The route /auth/invite/:token on the backend (still works with any code format)
Academy join codes (the separate joinCode field on academies table — different feature)
Tasks
Server — short code generator: In server/routes.ts, replace all instances of crypto.randomBytes(8).toString("hex") used for invite codes with a helper function that generates a 6-char uppercase alphanumeric code (chars A-Z, 0-9). Apply this to:

Player creation route (~line 7044)
/api/players/:id/invite GET route (~line 7109)
/api/players/:id/invite/regenerate POST route
Academy invite creation routes (wherever academyInvites inviteCode is generated)
Server — stop building invite URLs: In /api/players/:id/invite GET and regenerate endpoints, remove the invite URL construction code. Return only inviteCode (not inviteLink).

AdminPlayersScreen.tsx: Change the "Share this link" UI to show the short code prominently (large bold text, box style). Replace "Copy Link" with "Copy Code". Remove the URL display.

PlayersScreen.tsx: Same — replace URL display with prominent short code display and "Copy Code" button.

LoginScreen.tsx: Update the invite code entry screen:

Change label from "INVITE CODE OR LINK" to "INVITE CODE"
Remove "paste the full invite link" from the subtitle
Remove the URL-stripping logic (split("/").pop()) since there are no more URLs
Placeholder: "Enter your 6-character invite code"
Restart the server workflow.
