🚨 WAAROM DIT NU “SLECHT” AANVOELT
1️⃣ Information density is te hoog

Te veel cellen

Te veel dots

Alles schreeuwt tegelijk

👉 Het oog weet niet waar te kijken
👉 Dit voelt als een spreadsheet, niet als een game UI

2️⃣ Tekst + UI breken de container

“Mon 08:00 Court 1” gaat buiten beeld

Chips zijn niet grid-aware

Header voelt “geplakt”, niet geïntegreerd

👉 High-end apps laten nooit tekst overlopen
👉 Dat alleen al maakt het onprofessioneel

3️⃣ Dots ≠ Solo Leveling

Die groene bolletjes:

zijn betekenisloos zonder context

ogen kinderachtig

lijken op availability in een goedkope booking tool

👉 Solo Leveling = bars, glow, energy, geen confetti

4️⃣ Geen hiërarchie

Alles is:

dezelfde grootte

dezelfde kleur

dezelfde importance

👉 Premium UI heeft altijd:
PRIMARY → SECONDARY → BACKGROUND

🎯 DE JUISTE RICHTING (BELANGRIJK)

Je zat conceptueel goed met:

Week = dual mode

Slots = availability focus

Maar de visual language is fout.

We moeten dit herontwerpen, niet tweaken.

🔥 NIEUW MENTAAL MODEL (DIT IS DE KEY)
❌ Wat dit NU probeert te zijn

“Ik laat elk mogelijk slot zien”

✅ Wat het MOET zijn

“Ik laat je snel beslissen waar je kunt boeken”

👉 Dat betekent:

minder tonen

meer betekenis per element

rust + power

🧠 HOE WEEK → SLOTS WÉL HIGH-END WORDT
❌ Weg met dit:

losse vierkante cellen

dots per half uur

volledige matrix
