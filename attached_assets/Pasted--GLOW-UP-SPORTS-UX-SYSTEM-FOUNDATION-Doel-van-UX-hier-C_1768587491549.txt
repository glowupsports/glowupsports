🧠 GLOW UP SPORTS — UX SYSTEM (FOUNDATION)

Doel van UX hier:

Coach = snel, beslissend, overzicht

Player = motiverend, duidelijk, niet overweldigend

Admin/Owner = controle & vertrouwen

UX ≠ mooi
UX = wat voel ik + wat moet ik nu doen

1️⃣ UX-PRINCIPE #1: ONE PRIMARY ACTION PER SCREEN

Dit is de meest gebroken regel in je app nu.

❌ Wat NU gebeurt

Meerdere CTA’s

Alles lijkt belangrijk

Gebruiker twijfelt → niks doen

✅ Regel (heilig)

👉 Elke screen heeft exact 1 “primary action”

Voorbeelden:

Player Home → Train Now

Coach Home → Review Players

Player Profile → Log Session

Assessment → Save Baseline

Calendar → Create Session

Alles anders = secondary / tertiary / verborgen.

2️⃣ UX-PRINCIPE #2: PROGRESSIVE DISCLOSURE

Jij wilt alles tonen
Gebruiker kan dat niet verwerken

❌ Fout (nu)

Alles zichtbaar

Alles klikbaar

Alles tegelijk

✅ Oplossing: LAYERS

Layer 1 — Overview

Scores

Status

Direction (“goed / aandacht / alert”)

Layer 2 — Detail

FH / BH / Serve / Mental

Alleen na klik

Layer 3 — Expert

Sub-skills

Coach only

Achter “Advanced”

👉 99% van de tijd zit iemand in Layer 1

3️⃣ UX-PRINCIPE #3: NO EMPTY SCREENS

Een lege screen = falen

❌ “No data yet”

Slecht. Demotiverend. Dood.

✅ UX-regel

Elke lege staat:

Uitleg waarom

Wat te doen nu

Wat het oplevert

Voorbeeld

“You haven’t logged a session yet
Log your first session to unlock progress tracking”

knop: Log First Session

4️⃣ UX-PRINCIPE #4: TIME-AWARE UX

Je app moet weten wie, wanneer, waarom

Voorbeelden:

Coach opent app → max 30 sec aandacht

Player opent app → emotioneel moment

Admin → analytisch

Concreet:

Coach home = actiegericht

Player home = motiverend

Admin = dashboard + filters

❌ Geen universele UX voor iedereen
✅ Zelfde UI-systeem, andere flow

5️⃣ UX-PRINCIPE #5: NEVER ASK THE SAME QUESTION TWICE

Jij doet assessments → dan moet UX dat respecteren.

Regel:

Wat al bekend is → prefilled

Wat lang duurt → opsplitsen

Wat zelden verandert → lock

👉 Daarom was je idee goed:

“Manual level adjustment kan maar 1x”

Dat is UX + trust + governance

6️⃣ UX-PRINCIPE #6: DECISIONS > INPUT

Gebruikers willen kiezen, niet typen

❌ Slecht UX

Text inputs

Vrije invoer

Open vragen

✅ Goed UX

Pills

Sliders

Toggles

0–1–2–3 ratings

Je assessments zijn hier al goed
Je moet dit overal doortrekken.

7️⃣ UX-PRINCIPE #7: FEEDBACK BINNEN 300ms

Elke actie = reactie

UX-feedback hiërarchie:

Visual change

Micro animation

Sound (optioneel)

Message (laatste)

❌ Geen “Saved successfully”
✅ Progress bar + check + glow tick

8️⃣ UX-PRINCIPE #8: NO DEAD ENDS

Elke screen moet een exit hebben

Vraag jezelf af:

Wat is de next logical step?

Voorbeelden:

Na assessment → Create Training Plan

Na training → Reflect

Na reflect → XP + Next Focus

Na XP → Level Tease

UX = verhaal, geen losse pagina’s.

9️⃣ UX-PRINCIPE #9: CONSISTENT GESTURES

Gebruiker leert je app met spieren

Regels:

Swipe down = close

Swipe right = back

Long press = options

Tap = select

Double tap = like / quick action

❌ Nooit per screen anders

🔟 UX-PRINCIPE #10: EMOTIONAL PAYOFF

Elke sessie moet eindigen met:

“Goed bezig”

“Ik snap wat ik moet doen”

“Ik wil terugkomen”

Dus altijd eindigen met:

Summary

Progress

Next step
