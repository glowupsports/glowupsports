🎯 Probleem dat je voelt (terecht)

Je wilt niet handmatig elke keer:

"versionCode": 2 → 3 → 4 → 5


Want:

je vergeet het een keer ❌

Play Store weigert je build ❌

frustratie ❌

✅ Oplossing: EAS doet dit AUTOMATISCH voor je
🔥 Beste oplossing (aanrader)

Laat Expo/EAS de versionCode zelf verhogen.

Stap 1 — VERWIJDER versionCode uit app.json

👉 Dit is belangrijk.

❌ HUIDIG:

"android": {
  "package": "com.glowupsports.app",
  "versionCode": 2
}


✅ DOE DIT:

"android": {
  "package": "com.glowupsports.app"
}

Stap 2 — Zet auto-increment aan in eas.json

Open (of maak) eas.json:

{
  "build": {
    "preview": {
      "android": {
        "autoIncrement": true
      }
    },
    "production": {
      "android": {
        "autoIncrement": true
      }
    }
  }
}


📌 Wat gebeurt er nu:

Elke EAS build → versionCode +1

Geen handwerk meer

Play Store happy ✅

📱 Hoe zit het met version (1.1.0)?

Belangrijk verschil:

Veld	Betekenis
version	Wat de USER ziet (1.1.0)
versionCode	Interne Android teller

👉 version moet je nog zelf aanpassen
👉 versionCode doet EAS automatisch

Best practice:

Bugfix → 1.1.1

Feature → 1.2.0

Grote update → 2.0.0
